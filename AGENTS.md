# Tickr — agent conventions (binding)

Native macOS 14+ stock monitor. SwiftUI + Swift Charts. No third-party dependencies
unless the issue explicitly says so.

## Layout (two-target split — respect it)
- `TickrCore/` — SPM package: models, `QuoteProvider` protocol + implementations,
  favorites store, scheduling. **Platform-agnostic: importing SwiftUI or AppKit here
  is a defect.** Testable with `swift test` anywhere, including Linux.
- `Tickr/` — the SwiftUI app target. UI only; all logic lives in TickrCore.
- `TickrTests/` — app-target unit tests (XCTest).

## Hard rules
- TickrCore public API is consumed by app-target code you may not be able to see or
  build (the Codex sandbox has no macOS toolchain). Changes to EXISTING public
  protocols/types must be additive-only via new protocols or extensions, unless the
  issue explicitly authorizes a breaking change (PR #8 incident, 2026-07-10).
- `Tickr.xcodeproj` is GENERATED by XcodeGen and never committed. Source of truth is
  `project.yml`, which agents must NOT edit (operator-only). New source files under
  `Tickr/`, `TickrTests/`, or the package are picked up automatically.
- Never touch `.github/`, `AGENTS.md`, `Tickr/Tickr.entitlements`, `.claude*`.
- Tests are required for every change. Tests never hit the network — use fixtures
  and the mock provider.
- No force-unwraps in production code. Swift Concurrency (async/await), not callbacks.
- API keys never appear in the repo, in code, or in fixtures.

## Check commands (run what your environment supports; CI runs all)
1. `swift test --package-path TickrCore`   ← always run this
2. `xcodegen generate && xcodebuild -project Tickr.xcodeproj -scheme Tickr test -destination platform=macOS CODE_SIGN_IDENTITY=- CODE_SIGNING_REQUIRED=NO`
   ← REQUIRED before finishing: Xcode is installed in this environment (26.6+).
   CI re-verifies, but a diff that was never built locally is a defect.
